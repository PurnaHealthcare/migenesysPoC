## 1. Purpose

This SOP defines the standardized processes, user flows, data handling principles, and design guidelines for a cross-platform healthcare application that enables users to:

* Securely manage personal health information
* Track treatments in a **welcoming, non-clinical way**
* Maintain a lifelong medication and medical report history
* Receive smart medication reminders
* Use the app comfortably multiple times per day
* Access the app in multiple languages and regions
* Experience minimal cognitive and data-entry burden

---

## 2. Guiding Principles

### 2.1 Minimum User Effort

* Users should **never enter data the system can infer**
* Manual input is limited to **choices and confirmations**

### 2.2 Welcoming & Self-Initiating Language

* Avoid fear-based or clinical terms
* Use **â€œHealth Journeyâ€** instead of illness/disease
* Encourage care, not diagnosis

### 2.3 Progressive Disclosure

* Show only what is needed at the moment
* Advanced fields appear only if required

### 2.4 Privacy by Default

* All user data is private and visible only to the user
* No sharing unless explicitly initiated by the user

## 3. User Roles

### 3.1 End User

* Sole owner of all personal health data
* Can create, edit, view, export, or delete their data
* No other users or administrators can view PHI
* **App Name**: MiGenesys Life

### 3.2 Physician (DocAssist)
* Invited by the Patient (End User)
* Can view specific Health Journeys shared by the patient
* Can prescribe medications and add notes to the journey
* **App Name**: DocAssist

### 3.3 Organization (MiGenesys Care)
* Healthcare Organization or Clinic
* Manages list of Physicians
* View aggregated (anonymized) statistics or patient-specific data *only if explicitly shared*
* **App Name**: MiGenesys Care

### 3.4 Distributor/Reseller (MiGenesys Partner)
* Manages B2B relationship with MiGenesys
* Handles inventory, licensing, or distribution aspects
* **App Name**: MiGenesys Partner
